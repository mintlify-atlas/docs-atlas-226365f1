Your colleague quit, leaving behind unmaintained docs. Your intern left with an unfinished project. Your mentor graduated, taking all their context with them.

**colleague.skill** lets you distill a real person's work patterns and personality into an AI skill that thinks, codes, and communicates just like them — using their Feishu messages, DingTalk docs, Slack threads, emails, and more.

## How it works

<Steps>
  <Step title="Install the skill">
    Clone the repo into your Claude Code skills directory — the whole repo is the skill.

    ```bash
    git clone https://github.com/titanwings/colleague-skill ~/.claude/skills/create-colleague
    ```
  </Step>
  <Step title="Run /create-colleague">
    In Claude Code, type `/create-colleague`. Answer 3 quick questions: alias, company/level/role, and personality tags. All fields except the alias can be skipped.
  </Step>
  <Step title="Provide source material">
    Choose how to import data — auto-collect from Feishu/DingTalk/Slack, upload files, or paste text directly. More material means a more accurate skill.
  </Step>
  <Step title="Invoke your colleague">
    Once created, call the skill with `/{slug}`. It responds with their technical standards, their communication style, and yes — even their tendency to shift blame.
  </Step>
</Steps>

## What gets generated

Each colleague skill has two parts that work together:

| Part | What it captures |
|------|-----------------|
| **Work Skill** | Technical standards, system ownership, code review preferences, workflows, and accumulated experience |
| **Persona** | 5-layer personality: hard rules → identity → expression style → decision patterns → interpersonal behavior |

<Tip>
  The more raw material you provide — especially long-form writing and decision-making replies — the more accurate the generated skill will be.
</Tip>
